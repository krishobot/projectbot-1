## Why this combination

The two motions (running an agency + building a SaaS) are wildly different — different customers, different cadences, different brain shapes. But the same human runs both, and the same brain holds both. The bundle is the cleanest way to:

1. **Keep client work separate from product work.** agency-pack writes to `brain/companies/`, `brain/deals/`, `brain/reports/`. technical-founder-pack writes to `brain/decisions/`, `brain/feedback/`, `brain/changelog/`. No collisions.
2. **Reuse the same astack workspace + tbrain backend.** One install. One Discord. One set of cron jobs.
3. **Cross-pollinate where it helps.** A pattern that emerges from your agency client work (e.g., "agencies are paying for this kind of audit") becomes a `brain/feedback/` instance in the SaaS roadmap. The brain notices.

If you only run one of these motions, buy the relevant single pack. If you run both — even casually — the bundle is genuinely the right shape.
